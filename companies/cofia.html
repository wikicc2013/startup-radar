产品
自我编写的 AI 自动化平台。安装一个轻量桌面 agent，被动观测系统事件（应用切换、UI 交互、剪贴板、匿名化网络流量/API 调用），用序列模式挖掘（如 PrefixSpan/SPADE 类算法叠加 LLM 语义理解）识别可重复工作流，自动生成可执行的自动化 agent——无需用户写 prompt、无需搭流程、无需描述自己的工作。生成的自动化经人工审核后启用。典型场景：搭建潜客名单、外联与跟进、招聘管道维护、数据报表生成。
商业模式
B2B SaaS，按席位定价（seat-based，尚未公开价格）。目标人群为运营、销售、招聘等含大量跨应用重复点击的团队。企业侧在建 SOC2 Type II、GDPR DPA、代码签名与公证的桌面应用、MDM 部署兼容、IT 实时审计面板等合规能力。
竞争格局
与 Zapier/Make 等『用户手动搭流程』的自动化工具正面区隔——Cofia 主打『观察你实际做什么后自动产出你没想到要搭的自动化』。护城河更多在分发信任（说服企业 IT/法务在员工电脑装监控软件）与积累的工作流模式语料库（12+ 个月客户数据后形成冷启动壁垒），技术栈本身 4-6 个月可被复刻。与 RPA（UiPath 等）、桌面 agent/computer-use 类产品在能力上有重叠。

对制造业客户意味着什么

制造业办公与业务后台（采购、供应链协同、质量报表、订单跟进、招聘）存在大量跨 ERP/MES/OA/邮件的重复点击流。Cofia 式『被动观测→自动发现→生成自动化』的范式，理论上能替代『先请顾问梳理流程再上 RPA』的重投入路径，尤其适合流程未标准化、说不清自己在做什么的一线业务岗。落地前提是企业允许在办公终端部署观测 agent，制造企业对数据外发普遍敏感，私有化/内网部署将是刚性门槛。

🇨🇳

国内对标厂商

影刀 RPA / UiBot（来也）——桌面级流程自动化，但仍以人工录制/搭建流程为主、实在智能 TARS / 来也科技——RPA + Agent 融合方向、字节『扣子/Coze』、腾讯元器等 Agent 编排平台——低代码搭建 vs Cofia 的自动发现、钉钉/飞书的流程自动化与 AI 助手——观测协同数据生成自动化的思路可类比

国内对标集中在影刀、来也、实在智能等 RPA 厂商，以及扣子/元器等 Agent 平台，但主流仍是『人工搭流程』；Cofia 的差异点『自动发现该自动化什么』在国内尚无成熟对应产品，可作为观察窗口。
🧩

可迁移的干法 / 应进场景词典的词条

流程自动化、RPA、AI Agent、桌面观测/行为挖掘、工作流发现、运营自动化、销售外联自动化、招聘流程自动化、无提示自动化

可迁移打法：1）用桌面/网络事件被动挖掘高频重复工作流，把『识别自动化机会』这一最难的环节交给系统；2）生成的自动化先经人工审核再上线，降低信任门槛；3）以运营/销售/招聘等高重复岗位切入，按席位收费；4）用积累的工作流模式语料形成数据壁垒。制造场景可先在单一高重复岗（如采购对账、报表生成）做私有化试点。
⚠️

风险与存疑点

1）隐私与信任是命门，任何数据事故会直接击穿产品前提；2）冷启动——需足够观测时长才出建议，早期留存风险高；3）技术可复刻性高（4-6 个月），真正壁垒需 18-24 个月数据沉淀；4）企业侧最难的不是工程而是说服 IT/法务允许安装监控软件；5）在中国需私有化+合规改造，直接照搬 SaaS 模式不可行。
