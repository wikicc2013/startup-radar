## 转型有术判断
1. **对制造业客户意味着什么**：制造业岗位工作流‘动态多变、静态 Agent 易卡死’这一痛点与 Theta 的定位高度吻合。可对标的落地面：把排产、采购寻源、质量判定、供应链/库存调拨等重复但情境敏感的岗位，先‘映射工作流+定义成功标准’，再在仿真环境里用一方历史数据训练专家级 Agent，边跑边学。‘记忆层让 Agent 跨 run 累积经验’这一点，正对应制造现场‘老师傅经验难沉淀、难复制’的老问题。
2. **国内对标厂商**：面壁智能/各‘数字员工’与岗位 Agent 平台（专用岗位 Agent 方向）、澜码科技（企业级 Agent 平台，面向业务流程）、实在智能 / 来也科技（RPA→Agent，数字员工方向）、记忆层/Agent 基础设施：国内尚以 Mem0 类开源方案二次开发为主，专注‘Agent 记忆层’的独立创业标的稀缺
   > 国内‘数字员工’多停留在 RPA+LLM 编排层，缺的正是 Theta 强调的‘从每次执行中自学习、跨 run 变强’的记忆与 RL 闭环；‘工作流→仿真环境→专家 Agent’的训练范式在国内制造场景尚属空白，是可迁移的差异化打法。
3. **可迁移的干法 / 应进场景词典的词条**：企业专用AI智能体、Agent记忆层/自学习、工作流仿真环境训练、数字员工、后台运营自动化、财务建模、供应链/库存、强化学习/post-training、前置部署(forward-deployed)
   > 可迁移三步法：①对一个具体岗位做工作流拆解并定义可量化的成功标准；②用企业一方历史数据+专家知识构建带反馈闭环的仿真/评测环境；③forward-deployed 团队与业务并肩部署，让 Agent 在真实+仿真数据上持续自学习。关键抓手是‘先建 eval/环境、再谈自动化率’，避免一步到位全托付。
