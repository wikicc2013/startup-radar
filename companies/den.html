对制造业客户意味着什么

Den 的价值不在制造业专用功能，而在它押注的一条通用范式：把「搭建并指挥 AI agent」这件事从工程师手里下放给普通业务人员，载体是人人会用的 Slack/Notion 式界面。对制造企业而言，最痛的从来不是缺算法，而是采购、计划、质量、售后这些一线岗位的重复流程无人能低成本自动化——IT 排期长、外包贵、业务自己不会写代码。Den 这类「知识工作者版 Cursor + agent 市场」提供了一种设想：让排产员、质量工程师、供应链专员用自然语言描述自己的重复动作（催料、对账、生成 8D 报告、跟单），生成能实时协作、下班后仍在跑的 agent，并在市场里复用同行沉淀好的工作流模板。它把制造业 AI 落地的瓶颈从「找供应商做项目」变成「业务自助搭工具」，这正是制造企业 AI 规模化最需要跨过的那道坎。

中国市场 Den 这一层被两类玩家挤满：一是钉钉/飞书/企业微信三大协作平台自带 AI agent 能力，天然占住「Slack/Notion 载体 + 非工程师搭 agent」的入口；二是扣子/澜码等专注 agent 搭建与市场的创业公司。差异在于 Den 想把「协作载体」和「agent 构建器 + 市场」捏成一个 AI 原生新工作空间，而中国是把 AI 塞进既有超级 App。对制造企业选型，国内更现实的路径是在已部署的钉钉/飞书上叠 agent，或用扣子搭轻量流程；Den 模式的启示是「一个自然语言界面统一生产力栈 + 可交易的工作流市场」，这个产品形态国内尚无完全对位者。

可迁移的干法 / 应进场景词典的词条

非工程师自助搭 agent、自然语言统一生产力栈、AI 原生协作工作空间、agent 工作流市场、人机实时协作/下班后仍在跑、agent 集群编排（swarm）

可迁移到制造业的干法：① 把一线重复流程（催料、对账、跟单、质量报告生成）交给业务人员用自然语言自建 agent，绕开 IT 排期；② 建企业内部「agent 工作流市场」，让一个工厂验证过的排产/质检 agent 模板在集团多厂复用，把个人经验沉淀成可交易资产；③ agent 下班后持续跑（夜间对账、跨时区跟单、异常巡检），把人力从值守中解放；④ 用协作工作空间做人机同框，让 agent 像同事一样在群里被 @、交办、审核，降低一线接受门槛。核心迁移点是「让不会写代码的业务一号位成为 agent 的构建者与指挥者」。
