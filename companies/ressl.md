## 转型有术判断
1. **对制造业客户意味着什么**：Ressl 真正卖点不是'又一个 AI 客服'，而是'不换系统、直接吃掉胶水工工作'的落地路径——Agent 骑在 ServiceTitan/QuickBooks 等老旧无 API 系统之上，用点击/复制粘贴替人操作。对制造业直接映射：车间与供应链里同样存在大量'在 ERP/MES/供应商门户之间搬数据'的胶水岗(订单录入、报价、催料、对账开票)。客单 $40-50K/年、上线数周即产生收入，证明存量系统林立的中小企业里贴着旧系统跑是可行商业模式。
2. **国内对标厂商**：神策/售后宝等家服/售后 SaaS 中的 AI 派单调度、容联云/百应/追一等 AI 外呼与智能客服、汇联易/合思等费控开票自动化、国内暂无专门面向'蓝领技工小企业、架在现有工单系统上的 AI 员工'的成熟玩家，属空白位
   > 国内在单点能力(AI 外呼、智能派单、费控开票)有成熟供应商，但缺一个把这些串起来、以'AI 员工替代一个行政人头'为交付单元、瞄准蓝领小微企业的整合者，属对标缺口与机会位。
3. **可迁移的干法 / 应进场景词典的词条**：蓝领技工数字化、家庭服务(HVAC/水暖/电工)、AI 员工/数字员工、线索即时响应与自动排期、自动报价与投标、采购与配件调度、后台开票与工单跟进、架在存量系统之上(无迁移)、私募 roll-up 赋能
   > ①'不迁移、骑存量系统'降低中小客户上手门槛，绕过换系统阻力；②按'替代一个人力头寸'定价(对标 $60K 行政人力)，把 ROI 讲成人力替代而非工具订阅；③从'线索响应→报价→采购→开票'现金流链条切入，先窄后扩。制造业中小配套厂的销售内勤/采购内勤高度同构。
4. **风险与存疑点**：①仅 pre-seed $500K、团队 3-4 人、2025 成立，极早期；②骑无 API 老系统靠 UI 自动化，脆性高、易被上游改版打断；③无公开具名客户，'数周内数万美元+三位数周增长'为自述；④ServiceTitan 等系统方自建同类 Agent 则被平台吞并；⑤融资数字各源不一($500K/$750K)。
