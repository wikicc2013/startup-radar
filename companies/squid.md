## 转型有术判断
1. **对制造业客户意味着什么**：Squid 的内核是「把散落在邮件/表格/PDF/多版本模型里的领域决策，收敛成一个带版本、带证据、可协作的实时数字模型」，这一范式对制造业高度可迁移：工艺参数、BOM 变更、产线布局、设备台账同样长期困在 Excel 与多份不一致的图纸/模型里。制造企业可借鉴其「单一可信模型 + 版本控制 + 证据溯源 + AI agent 驱动工作流」思路，用于工厂数字孪生、工艺变更管理（ECN）、设备资产建模等场景。
2. **国内对标厂商**：国电南瑞（电网自动化/调度与规划软件龙头）、远光软件（电力企业数字化）、朗新科技（能源数字化/电力营销与用能服务）、东方电子（电网自动化）、恒华科技（电网规划设计与 GIS 平台）
   > 国内电网规划/能源数字化由国电南瑞、远光软件、朗新、恒华、东方电子等占据，多为项目制、重实施的电力信息化软件；Squid 走的是轻量浏览器化、AI agent 化、产品化 SaaS 路线，与国内「大集成商 + 定制交付」模式形成对照。国内对标企业若要接住电力电子/电网电气设备制造客户的数字化诉求，可参考其产品化与协作层打法。
3. **可迁移的干法 / 应进场景词典的词条**：电网规划、能源数字化、配电网/输电网建模、并网接入管理、GIS 电网数据统一、版本化数字孪生、AI Agent 工作流、公用事业 SaaS
   > ① 创始人从行业内部（国家电网、Octopus）出来，用「亲历痛点」建立信任并快速拿下前雇主生态内的首个 DSO 合作；② 不替代既有分析工具，而做其上的统一协作/真源层，降低替换门槛；③ 以「实时+版本+证据挂载」作为差异化，把合规与可审计变成卖点（宣称有 SOC 2/ISO 27001 方向，待官方证实）。制造业软件出海或服务大客户时可复用「行业老兵 + 协作层切入 + 可审计真源」组合拳。
4. **风险与存疑点**：① 团队仅 3 人、2026 年成立，尚极早期，除国家电网 DSO 的 FlexPortal 合作外无公开付费客户与营收数据；② 融资金额、投资方（除 YC 标准投资外）均未披露；③ 官网提及的 SOC 2/ISO 27001 认证与部分证言在权威页面未获交叉确认，置信度偏低；④ 电网软件销售周期长、监管重、替换成本高，早期 SaaS 打法能否规模化存疑。
