- **地点**：美国加州旧金山（San Francisco, CA）
- **产品**：面向硬件/物理工程的 AI 智能体平台，类 Cursor 的人机协作界面，让工程师与 AI Agent 共同处理多模态测试数据（遥测/telemetry、仿真、视频、工程文档、测试标准）。可连接文件、数据库、API 及 MATLAB、Adams、VI-Grade 等工具，自动跨整个测试活动分析失效、从数据库拉取 KPI、检验温度/载荷等工况对性能与可靠性的影响，并生成报告与看板。宣称把原本数小时到数天的硬件工程任务压缩到几分钟、让工程师处理 100x 数据。四大能力：Connect（接入数据源）、Collaborate（团队共享工作区）、Encode（把团队方法论沉淀为可复用、可版本化的工作流）、Govern（管理面板、审计日志、成本追踪等企业级管控）。
- **商业模式**：企业级 SaaS，基于额度（credit-based）的用量计费与按团队预算管控；已通过 SOC 2 Type I 审计、端到端加密。从赛车运动（motorsports）时间敏感、销售周期短的场景切入，再向制造业、机器人等扩张。官方称物理世界测试（开发/耐久/验证）是约 2500 亿美元市场。
- **竞争格局**：硬件测试数据分析/工程知识自动化赛道。传统对手为遗留测试数据管理与后处理工具链（如各类 telemetry 后处理、MATLAB 脚本、Adams/VI-Grade 仿真后处理、PLM/测试数据管理系统）及工程师手工分析；新兴对手为面向工程数据的 AI copilot。差异化在于把赛车级实时遥测分析经验做成 Agent 化的多模态分析平台。

## 转型有术判断
1. **对制造业客户意味着什么**：MOVEdot 的核心价值主张——把资深工程师的『看数据—找关联—定失效根因—写报告』全链路，从数天压到数分钟，并让团队用 10-100x 的数据做决策——直接对应制造业研发/试验环节的最大痛点：测试数据爆炸、后处理靠人肉脚本、经验只在少数老工程师脑子里。对整车、动力电池、家电、装备等有大量台架/耐久/可靠性试验的企业，这是一个『试验数据提效 + 工程知识资产化』的模板：Agent 跨整个测试活动做失效分析、按工况（温度/载荷/振动）自动交叉验证、并把团队方法论固化成可版本化工作流，缓解老师傅退休导致的知识流失。
2. **国内对标厂商**：硬件研发/试验数据分析：国内主机厂与零部件企业的试验场/台架测试数据后处理平台（多为内部自研或系统集成商定制）、EDA/仿真后处理：国产 CAE 与仿真数据管理厂商（如云道智造/天洑软件等仿真与数据智能方向）、PLM/研发数据管理：数字化研发平台厂商（如用友、鼎捷、华天软件 PLM、思普等）承载测试/试验数据治理、工业时序/遥测数据分析：工业互联网与时序数据平台（如 TDengine 时序库 + 上层分析）在整车/装备测试场景的应用
   > 国内尚无与 MOVEdot 完全对位的『多模态硬件试验数据 AI Agent』独立产品，能力分散在三处：一是 PLM/研发数据管理厂商的试验数据治理模块，二是 CAE/仿真后处理与工业时序数据平台，三是主机厂/大院所的内部自研后处理脚本与数据平台。空白点在于把遥测/试验/仿真/文档多模态数据 + LLM Agent + 可复用工作流打通的产品化层，这正是国内可切入的窗口。
3. **可迁移的干法 / 应进场景词典的词条**：硬件测试数据分析、试验/台架数据后处理、遥测telemetry分析、失效分析/根因定位、多模态工程数据（数据+视频+文档）、研发方法论沉淀为可复用工作流、赛车运动motorsports工程、自动生成测试报告与看板、工程AI Agent/copilot
   > 可迁移打法有三条：①『时间敏感高价值场景切入』——MOVEdot 从赛车（决策以小时计、销售周期短、首单两周内成交约 3 万美元）起步再横向扩张，国内可对标从赛车/新能源三电耐久/风电齿轮箱等『停机成本高、试验密集』的细分先立标杆；②『前向部署工程师转产品』——两位创始人把 8 年 FDE 经验直接编码进 Agent，国内做工业 AI 的团队同样应把行业老工程师的隐性经验做成 Encode 可复用工作流；③『Govern 先行』——一开始就上 SOC2、审计日志、成本追踪、按团队预算管控，降低大型制造企业采购与合规门槛。
