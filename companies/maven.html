产品
面向语音 AI 智能体的支付基础设施。当语音 Agent 在通话中需要收款时，通过一次 API 调用把来电者转接到 Maven（或用 SIP URI），由 Maven 以语音/拨号盘方式或短信 checkout 链接安全采集卡号、完成扣款，卡数据不经过客户服务器（PCI 合规由 Maven 承担），支付完成后自动把来电转回原 Agent。功能含卡号令牌化、失败重试、多语言、多网关。语音平台侧集成 VAPI、Retell、LiveKit、Twilio；支付网关侧集成 Stripe、Authorize.net、Adyen、Braintree、Shift4 等。
商业模式
按会话计费（Pay per session），无月费，提供免费额度（$0 起）。定位为开发者可自助接入的支付 API 层，替客户免建自有 PCI 合规基础设施。具体单价费率未披露。
竞争格局
同处 YC W26 及 Agent 金融基础设施赛道的相邻公司包括 Sponge（Agent 经济金融基础设施）、Orthogonal（面向 API 的 agentic payments）。上游支付网关本身（Stripe 等）为潜在既合作又竞争关系；传统电话收款方案为 IVR/人工坐席 + PCI 托管服务商。

对制造业客户意味着什么

制造业本身不是 Maven 的直接场景，但其模式对制造企业的'服务化收款'环节有借鉴：售后服务、备件订购、设备租赁/按次计费、经销商电话下单等场景中，若企业上了语音 AI 外呼/客服，Maven 式'通话中一键安全收款'可把'咨询-报价-成交'闭环压缩到一通电话内完成，减少人工坐席读卡与合规负担。

🇨🇳

国内对标厂商

语音外呼/语音客服 Agent 侧：容联云、思必驰、百应科技、云蝠智能等提供电话语音智能体，但普遍不自带 PCI 级电话收款支付层、聚合支付 API 侧：Ping++、连连支付、汇付天下等提供多渠道支付聚合与 API，但未针对语音 Agent 通话内实时收款做产品化、Agent 支付探索侧：支付宝/蚂蚁在 agentic payments 方向的探索、微信支付生态；国内暂无与 Maven 完全对位的'语音 Agent 通话中一键收款'专用基础设施

国内制造企业若要复制，缺的不是支付通道（连连/汇付/Ping++ 已足够），而是'语音 Agent + 通话内 PCI 级收款'的产品化拼接；现实路径更可能是语音厂商（容联云/思必驰）与聚合支付（连连/汇付）联合，或走微信支付/支付宝的扫码/短信 checkout 链接绕开电话读卡的合规难点。
