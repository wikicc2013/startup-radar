团队
联合创始人 Magnus Müller（CEO）与 Gregor Žunič（Zunic），2024 年在苏黎世联邦理工（ETH Zurich）Student Project House 孵化起步。团队约 7 人，现总部旧金山（部分研发仍在苏黎世）。
成立/批次
2024
地点
美国旧金山（起源于瑞士苏黎世 ETH Zurich）
产品
开源 AI 浏览器自动化框架。核心思路是把网页界面解析成结构化文本（DOM/可交互元素抽取）喂给 LLM，让 Agent 像人一样打开页面、点击、输入、填表、提交，从而在任意网站上执行任务（登录、数据抓取、QA 测试、下单、CRM 集成等）。Python 库为主（MIT 许可），并提供托管云端产品（Browser Use Cloud / Harness、隐身浏览器、代理轮换、验证码处理、并行执行、持久会话）。也提供可被 Claude Code、Codex、Cursor 等 Agent 调用的 CLI/skill。
商业模式
开源+云的双层商业化：开源库免费（MIT，靠社区规模与开发者心智获客），托管云服务按订阅/用量收费（有 30 美元/月起的基础档），面向需要稳定基础设施（代理、反检测、并行、会话管理）的生产级用户。
竞争格局
直接对标 OpenAI Operator/CUA、Anthropic Claude Computer Use；同类开源/基建有 Steel Browser、Hyperbrowser/HyperAgent、Vercel agent-browser、Microsoft Magentic 等。差异化：走 DOM 结构化文本路线（相对纯视觉更省 token、更确定性），且以超高人气开源社区为护城河。据第三方评测在 WebVoyager 基准上报约 89.1%，处于第一梯队。

对制造业客户意味着什么

对制造业的意义在于「打通没有 API 的老系统」。制造企业普遍被大量只有网页界面、没有开放接口的供应商门户、海关/报关系统、经销商后台、招投标平台、政府申报系统困住，日常靠人工在浏览器里反复查单、录数、下载对账单、抓竞品报价。Browser Use 这类把网页变成 Agent 可操作对象的框架，让 AI 能像员工一样跨系统跑通「登录→查询→抄录→回填」的重复流程，等于给没有集成能力的遗留系统补了一层「AI 外挂接口」。对供应链协同、采购比价、售后工单录入、多平台铺货这类跨系统人工搬运工作，是可直接复用的自动化底座；但生产级落地要解决稳定性（页面改版即失效）、权限合规与审计留痕。

国内对标厂商

智谱 AutoGLM / Open-AutoGLM（GLM-4.5、GLM-4.5V 驱动的国产 GUI/浏览器 Agent，含 autoglm-browser-agent，走开源+自研模型路线，是最接近的国产对标）、通义/阿里系及各大厂的 GUI Agent 研究方向（RPA+大模型融合，多为闭源或企业内用）、国产 RPA 厂商（影刀、来也 UiBot、艺赛旗等）在向 LLM 驱动的智能 RPA 演进，与 Browser Use 在「网页流程自动化」上功能重叠，但技术路线偏录制/规则+大模型增强

国内最贴近的是智谱 AutoGLM 系列——同样开源、同样做浏览器/GUI 操作，且用纯国产模型驱动，主打手机+电脑全端；差异是智谱更偏消费级通用助理与自研模型闭环，Browser Use 更偏开发者基础设施与 LLM 无关（可接任意模型）。另一条对标线是国产 RPA 厂商的「智能化转型」：它们有大量制造业客户与流程落地经验，但历史路线是规则/录制式，正被 LLM 驱动的浏览器 Agent 从下方颠覆。对国内团队的启示：Browser Use 用「极致开源人气→云服务变现」的打法在开发者侧建立了心智，这是国产 RPA 与大厂 Agent 值得警惕的护城河构建方式。
🧩

可迁移的干法 / 应进场景词典的词条

浏览器自动化、Web Agent、无API系统集成、DOM结构化、跨系统数据搬运、开源社区飞轮、LLM无关基础设施

可迁移的干法有三条：一是「结构化优于截图」——把界面解析成文本喂大模型比纯视觉更省成本、更确定，制造业做单据/表单类自动化时值得优先尝试 DOM/文档结构化路线；二是「给遗留系统补 AI 接口」——不改造老系统，用浏览器 Agent 在最上层跑通人工流程，是低改造成本切入制造业信息化死角的现实路径；三是「开源飞轮做基础设施」——用开源项目快速聚拢开发者与真实场景反馈，再用托管云（稳定性/合规/并行）变现，是工具型创业沉淀护城河的范式。
⚠️

风险与存疑点

风险①脆弱性：依赖页面结构，目标网站一改版流程即断，生产级稳定性与维护成本高，且反爬/反自动化对抗持续升级（隐身浏览器、验证码是长期军备竞赛）。风险②巨头夹击：OpenAI Operator、Anthropic Computer Use、Google 等把浏览器/电脑操作能力直接内建进模型层，开源框架的差异化窗口可能被压缩，商业化云服务须证明比大厂原生方案更值得付费。风险③合规与安全：代表用户登录、抓数、下单涉及凭据处理、越权操作、验证码规避与各站点 ToS 合规争议，企业采用时的审计、权限与责任边界尚不成熟，是落地的主要阻力。
