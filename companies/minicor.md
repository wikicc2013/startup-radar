## 转型有术判断
1. **对制造业客户意味着什么**：制造业普遍存在大量『换不动』的老旧系统——车间 MES、老 ERP（含各种非标定制/停止支持版本）、供应链的 SAP/HighJump、以及一堆只有客户端界面、没有 API 的专用软件。Minicor 的价值主张恰是『不换系统、不等 API、把 AI 从 UI 层灌进去』：录一段人工操作即生成 API，AI 生成的数据（排产建议、质检结果、单据信息）通过自愈式桌面自动化写回系统。对制造业一号位的启示是——AI 落地最大的物理障碍常常不是模型，而是『最后一公里的系统对接』，而这一公里可以用桌面自动化 + 自愈 Agent 绕开高昂的系统改造。
2. **国内对标厂商**：来也科技（UiPath 式 RPA + IPA，遗留系统流程自动化）、影刀 RPA（易用录制式桌面自动化，中小企业遗留系统）、实在智能（TARS/Agentic RPA，屏幕理解 + 自愈式点击）、弘玑 Cyclone、艺赛旗（企业级 RPA，金融/制造遗留系统改造）
   > 国内已有成熟对标：来也、影刀、实在智能都在做『屏幕理解 + 录制 + 自愈点击』的桌面 RPA，且更贴近中国制造企业的老系统生态（用友/金蝶/各类国产 MES）。Minicor 的差异是定位于『卖给 AI 创业公司做集成层』并强调 API 端点化 + 合规（HIPAA/SOC2）——国内对标更多是直接卖给甲方做流程自动化。国内做制造遗留系统改造时，影刀/实在智能的『录制即用 + 自愈』模式最可直接借鉴。
3. **可迁移的干法 / 应进场景词典的词条**：遗留系统不换系统上AI、无API系统自动化、桌面RPA、自愈式自动化、AI数据回写系统、Windows虚拟机自动化、医疗EHR自动化、供应链SAP自动化、汽车DMS自动化、on-prem合规部署
   > 可迁移打法：① 以『录制人工操作视频 → 自动生成自动化』降低实施门槛，让业务人员而非集成商定义流程；② 确定性脚本打底 + AI Agent 只兜边缘情况，把成本压在低廉的确定性执行上、把 LLM 用在恢复环节，兼顾准确率与成本；③ 全流程录像回放 + 日志 + 告警，把『黑盒自动化』做成可审计可运维的产品，这是打进合规行业（医疗/金融/制造质量）的关键；④ on-prem 容器化跑在客户内网，解决数据不出厂的顾虑——制造业数据敏感场景可直接照搬。
4. **风险与存疑点**：① 目标仍是『新锐早期』（YC Spring 2026 / P26，成立 2023–24，约 7 人），无公开融资，商业化尚早；② 客户/traction 除『每周数千次工作流』外无具名披露，Mycroft/Pallet 仅从官网 logo 推测、置信度低；③ 桌面 UI 自动化天然脆弱，UI 大改版/权限变更/反自动化机制会影响稳定，虽宣称自愈但准确率 93–96% 仍意味约 4–7% 失败需人工兜底；④ 与经典 RPA 巨头及各家 computer-use Agent 正面竞争，护城河待验证；⑤ 目前重心在医疗 EHR，制造业为推断迁移场景而非其已披露主战场。
