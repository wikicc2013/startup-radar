产品
面向 AI 公司的『遗留桌面系统自动化』平台：客户用桌面客户端录一段人工操作视频，Minicor 即生成一个 API 端点；每次调用在 Windows 虚拟机上触发一次『自愈式』桌面自动化，把 AI 生成的数据读写进没有 API 的老旧桌面/客户端软件（EHR、ERP、DMS 等）。技术上把确定性 Python 自动化与 AI Agent（负责恢复与边缘情况）结合，宣称点击准确率 93–96%（纯 computer-use 方案约 80–85%）。支持 on-prem/云/Citrix 部署，含全流程录像回放、细粒度日志、Slack 告警，并提供 MCP 供编码 Agent 接入。已支持 Epic/Cerner/Athena/PS Suite（EHR）、Open Dental/Dental Vision（口腔 PMS）、CDK Global（汽车 DMS）、SAP/HighJump（供应链）、Wellsky/Home Care HomeBase（居家护理）。宣称 SOC 2 Type II、HIPAA 合规，on-prem 全容器化跑在客户网络内。
商业模式
用量计费——『只有数据成功写进去才付费』（Only pay when the data gets in），按成功执行的自动化次数收费；企业需 Book a Demo。定价未公开披露。
竞争格局
对标经典 RPA（UiPath/Automation Anywhere/Blue Prism）与纯 computer-use/桌面 Agent 方案；差异化在于把确定性脚本 + AI 自愈结合、以 API 端点形态卖给『要落地到客户遗留系统的 AI 创业公司』，主打上线快（POC 后 3 周上线，传统 RPA/集成常需 4 个月以上）与高点击准确率。

对制造业客户意味着什么

制造业普遍存在大量『换不动』的老旧系统——车间 MES、老 ERP（含各种非标定制/停止支持版本）、供应链的 SAP/HighJump、以及一堆只有客户端界面、没有 API 的专用软件。Minicor 的价值主张恰是『不换系统、不等 API、把 AI 从 UI 层灌进去』：录一段人工操作即生成 API，AI 生成的数据（排产建议、质检结果、单据信息）通过自愈式桌面自动化写回系统。对制造业一号位的启示是——AI 落地最大的物理障碍常常不是模型，而是『最后一公里的系统对接』，而这一公里可以用桌面自动化 + 自愈 Agent 绕开高昂的系统改造。

国内对标厂商

来也科技（UiPath 式 RPA + IPA，遗留系统流程自动化）、影刀 RPA（易用录制式桌面自动化，中小企业遗留系统）、实在智能（TARS/Agentic RPA，屏幕理解 + 自愈式点击）、弘玑 Cyclone、艺赛旗（企业级 RPA，金融/制造遗留系统改造）

国内已有成熟对标：来也、影刀、实在智能都在做『屏幕理解 + 录制 + 自愈点击』的桌面 RPA，且更贴近中国制造企业的老系统生态（用友/金蝶/各类国产 MES）。Minicor 的差异是定位于『卖给 AI 创业公司做集成层』并强调 API 端点化 + 合规（HIPAA/SOC2）——国内对标更多是直接卖给甲方做流程自动化。国内做制造遗留系统改造时，影刀/实在智能的『录制即用 + 自愈』模式最可直接借鉴。

可迁移的干法 / 应进场景词典的词条

遗留系统不换系统上AI、无API系统自动化、桌面RPA、自愈式自动化、AI数据回写系统、Windows虚拟机自动化、医疗EHR自动化、供应链SAP自动化、汽车DMS自动化、on-prem合规部署

可迁移打法：① 以『录制人工操作视频 → 自动生成自动化』降低实施门槛，让业务人员而非集成商定义流程；② 确定性脚本打底 + AI Agent 只兜边缘情况，把成本压在低廉的确定性执行上、把 LLM 用在恢复环节，兼顾准确率与成本；③ 全流程录像回放 + 日志 + 告警，把『黑盒自动化』做成可审计可运维的产品，这是打进合规行业（医疗/金融/制造质量）的关键；④ on-prem 容器化跑在客户内网，解决数据不出厂的顾虑——制造业数据敏感场景可直接照搬。
