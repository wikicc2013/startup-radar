产品
面向 AI 智能体的『人类专家即 API』。用户把 Humwork 的 MCP server 接入自己的 AI 智能体（如 Claude Code、Lovable、OpenClaw），当智能体卡壳（80% 完成后陷入同一 bug 死循环，或在合规、判断、审美上出错自己纠不了）时，通过 MCP 一键升级，30 秒内路由到一位经过验证的领域专家（资深工程师、律师、市场、设计师、科学家等）。专家直接与智能体对话（而非与用户），答案回灌进智能体上下文，用户不离开工作流即可继续。所有专家上线前经身份核验、技能测评、领域考核。
商业模式
双边实时市场（marketplace）。智能体端为专家咨询付费，Humwork 招募并验证领域专家、按次实时撮合供给。核心宣传语『AI agents will pay you to chat with them』——把人类专业能力做成智能体可调用的按需服务层/兜底升级路径。
竞争格局
定位为 AI 智能体的『专家升级层/HITP 兜底层』。相邻者包括：通用众包/专家网络（Upwork、Toptal、GLG 专家网络）、AI 数据标注与 RLHF 供给商（Scale AI、Surge AI、Mercor 侧重招募专家做训练/评估）、以及各智能体自带的人工 handoff 机制。差异点：不是给人做任务外包，而是让『智能体』在运行时实时呼叫真人专家、专家对话对象是 agent 本身，通过 MCP 标准协议接入。

对制造业客户意味着什么

制造业大量部署 AI 智能体（工艺参数问答、质检判定、设备故障诊断、供应链排程、合规文档生成）时，普遍面临『80% 能自动、剩 20% 卡死或不敢让 AI 自主决策』的问题。Humwork 这套『智能体运行时实时呼叫真人专家、答案回灌上下文』的模式，正对应制造场景里最需要的『人在最后一环兜底（HITP）』：例如产线诊断 agent 遇到罕见故障可 30 秒内升级到资深设备工程师，工艺 agent 拿不准的参数调整可升级到工艺专家，质量 agent 对边缘缺陷判定不确定时可升级到质检老师傅。把老师傅/工艺专家的经验做成智能体可按需调用的『专家 API』，是制造业隐性经验资产化的一条可行路径。

可迁移的干法 / 应进场景词典的词条

AI智能体人工兜底(HITP)、智能体运行时专家升级、MCP专家路由、专家众包/按需专家网络、coding agent debug 升级、AI 输出合规/审美人工把关、agent-to-human 实时对话

可迁移打法：1) 把『人工兜底』产品化为一个标准接口（MCP/API），让任何智能体一行接入就获得实时专家升级通道，而非各自造 handoff；2) 严格供给侧质量门槛——身份核验+技能测评+领域考核，用『验证过的专业人士』而非随机自由职业者建立信任，制造业尤其看重资质；3) 触发即升级——由 agent 自动判断卡壳并升级，而非等用户手动求助，把兜底做成默认安全网；4) 经验回灌——专家答案写回 agent 上下文并可沉淀为知识，长期把兜底数据转化为训练/知识库资产，逐步把 HITP 抬升为 HOTP。
