## 转型有术判断
1. **对制造业客户意味着什么**：把制造企业运营/采购/客服团队每周大量消耗在『定义清晰却从未自动化』的协调工作（催单、报价、PO 审批、异常升级）交给 Agent 执行，人只在需判断时介入。价值点在于不替换 ERP、直接骑在既有 SAP/Oracle/NetSuite 之上，落地周期从季度压到小时。
2. **国内对标厂商**：聚水潭/旺店通（订单-采购协同，偏电商）、黑湖智造（制造协同/工单）、简道云+ERP 集成类低代码工作流、用友/金蝶生态内的 AI Agent 插件方向
   > 国内黑湖、聚水潭做的是协同/订单中台，多为流程数字化与看板，尚未做到『自然语言描述即生成跨系统自主执行 Agent』；用友/金蝶在 ERP 生态内加 AI 助手但偏问答与单点动作。差距主要在 agentic 执行能力与多 ERP 通用连接层。
3. **可迁移的干法 / 应进场景词典的词条**：询价报价自动化(RFQ-to-Quote)、采购订单审批与供应商协调、订单履约异常监控、ERP工作流自主执行Agent、制造业后台协调自动化
   > ①选『定义清晰、纯手工、跨系统』的高频协调工作流（询价、催单、PO 审批）作楔子，ROI 直观；②骑在存量 ERP 之上不做替换，降低采购阻力；③自然语言配置+数小时上线，对标 SI 季度交付形成速度差；④长周期工作流的 checkpoint/断点续跑是制造场景的关键工程护城河。
4. **风险与存疑点**：①极早期（2026 成立、3 人、无公开客户与融资），一切以官方叙事为主；②官网 korsoai.com 主域抓取时返回『量化交易/ShepherdBeta』无关内容，疑为域名占位或缓存，需人工核对官网真实状态（消歧存疑点，但 YC 目录/Launch/多家媒体叙事一致，故仍锚定 YC 页）；③深度绑定多套 ERP 的通用连接层维护成本高，SAP/Oracle 自身 AI Agent 下场会挤压；④自主 Agent 在采购/报价涉钱环节的可靠性与责任边界是采纳门槛。
