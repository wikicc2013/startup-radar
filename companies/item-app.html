产品
item 是「AI 原生 / agentic CRM」，核心不是让人填表，而是让系统替人干活。三层结构：①Assistant——自然语言侧边助手，用一句 prompt 即可发跟进、调研线索、加录商机，无复杂 UI / 下拉框 / 字段；②Agents——把业务流程写成文档（类 Notion 文档）来「像培训新员工一样训练」智能体，让其自主跑销售、支持、客户成功等可重复流程；③Context/记录层——连接企业所有客户工具，自动采集并富集上下文（含自动 Web 调研公司/联系人信息），成为公司「唯一真源」，人与 AI 共享同一份完整视图。典型场景：外呼获客（在邮件与 LinkedIn 上找并联系合格线索）、入站线索资格判定与响应、存量客户增长（自动发个性化升级序列）、管道保温（据邮件与会议纪要自动更新商机）。含联系人/公司富档案、表格与看板视图。

对制造业客户意味着什么

对制造业/工业企业的营销·销售·客服，item 这类 agentic CRM 指向一个关键转变：CRM 从「销售人员事后手工填的台账」变成「自己干活、自己更新的执行系统」。制造业销售链条长（询盘→报价→打样→小批量→量产返单）、客户信息散落在邮件、微信/LinkedIn、ERP 和销售脑子里，传统 CRM 录入率极低、数据永远滞后。item 的思路——用邮件与会议纪要自动更新商机、用文档化流程训练 agent 跑外呼与存量客户升级——恰好命中工业企业「外贸业务员一人对几十个海外客户、跟进全靠记忆」的痛点：把标准化的报价跟进、样品催办、返单提醒交给 agent 自动执行，人只做关系与谈判等高价值判断。对制造业客服而言，入站询盘的资格判定与首轮响应可被 agent 承接，缩短对工业买家的响应时延。风险是制造业成交依赖线下信任与复杂技术沟通，agent 更适合「保温与前置筛选」而非替代关键谈判。

国内对标厂商

卫瓴科技（企微生态的协同 CRM，主打线索全生命周期与 AI 辅助跟进）、销售易 / 神州云动 CloudCC（本土 CRM，正加 AI Copilot）、纷享销客（连接型 CRM，面向 B2B 复杂销售，推 AI 销售助手）、探迹 / 探迹拓客（AI 获客 + 智能外呼，贴近 item 的外呼 agent 场景）

国内对标可分两类：一是「CRM 加 AI」的既有厂商——纷享销客、销售易、卫瓴，它们从记录/协同型 CRM 出发叠加 AI Copilot，路径与 item「原生 agent」相反（改造存量 vs 重建）；二是「AI 获客/外呼」新锐——探迹，直接对应 item 的外呼与线索富集 agent 能力。国内尚缺少「把整个 CRM 重构为自主 agent 执行系统」的原生玩家，多数仍停留在「助手/Copilot 嵌入」阶段，item 的「文档训练 agent + 唯一真源」范式对国内是可迁移的产品方向。

可迁移的干法 / 应进场景词典的词条

agentic-CRM、自主销售agent、外呼获客自动化、线索资格判定、管道自动更新、文档即流程训练

可迁移干法：①「文档即培训」——把销售 SOP、报价规则、跟进话术写成结构化文档喂给 agent，等于把老销售的隐性经验显性化、可复制，这对制造业「师徒制、经验难传承」的销售团队尤其有价值；②「记录层自动化」——不让人填 CRM，而是从邮件/会议纪要反向抽取更新商机，解决一切 CRM 落地的老大难（录入率低）；③「唯一真源 + 人机共视图」——人与 agent 共享同一份客户上下文，agent 自主执行、人做例外处理，是「Push Right + 人只在关键点介入」协同范式的典型落地。
