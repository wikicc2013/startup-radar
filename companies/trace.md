## 转型有术判断
1. **对制造业客户意味着什么**：制造企业后台（采购、供应商 onboarding、质量文档处理、订单履约、售后工单）同样存在大量跨系统重复 gruntwork，Trace『先建组织知识图谱→自然语言拆解为工作流→AI/人分工+审批门』的范式高度可迁移。对制造业尤其贴合的是：流程需人工兜底与合规审批（HITL 审批门），而非全自动黑箱，符合制造现场对可控性与可追溯的要求。
2. **国内对标厂商**：实在智能 / 澜码科技（Agent 编排 + 人机协同流程）、影刀 RPA、来也科技 UiBot（RPA 向 Agent 化演进，工作流自动化）、字节 扣子 Coze / 阿里 百炼（低代码 Agent 编排平台）、钉钉 / 飞书的 AI 工作流与审批流（组织内知识图谱 + 人机任务路由的对标场景）
   > 国内可对标『Agent 编排 + 人机协同』一线玩家（实在智能、澜码、影刀/来也的 Agent 化、扣子/百炼低代码编排、钉钉飞书 AI 审批流）。Trace 的差异化在于把『组织知识图谱作为共享上下文层』前置——这是国内多数 RPA/编排产品尚未夯实的一环，可作为制造企业选型时的评估维度。
3. **可迁移的干法 / 应进场景词典的词条**：AI Agent 编排、人机协同/任务路由、企业知识图谱/上下文层、后台流程自动化、自然语言转工作流、人工审批门 HITL、客户 onboarding、HR 简历筛选、文档处理、销售运营
   > 1) 连接企业现有系统（ERP/MES/OA/邮件/IM）自动抽取实体关系建图谱；2) 让业务人员用自然语言描述流程，自动生成可视化工作流并标注可自动化环节；3) 高价值/高风险环节设人工审批门，低价值重复环节交 agent；4) 用『活跃工作流数』『agent 承接任务占比（Trace 现约 10–14%）』作为落地度量，逐步提升自动化比例、量化人力 ROI。
4. **风险与存疑点**：1) 早期公司（2025 成立、$3M 种子、团队仅 2–3 人、30+ 客户），产品与稳定性未经规模验证；2) 无具名标杆客户与公开 ROI 数据，『降人力成本』为客户自述、未第三方证实；3) 依赖第三方垂直 agent 生态，编排层价值取决于生态成熟度；4) 与 Anthropic、Atlassian 等巨头的企业 agent 正面竞争，护城河待观察；5) 深度接入企业内部工具→数据安全与合规是制造/大型企业采用的关键门槛。
