## 转型有术判断
1. **对制造业客户意味着什么**：对制造业/工业企业的营销·销售·客服意味着：工业品（设备、软件、SaaS 平台、工业自动化方案）销售链条长、演示专业门槛高、跨时区跨语言、对销售工程师（SE）依赖重，恰是这类『会实操产品的 AI demo 智能体』最能压缩成本的场景。制造业官网询盘往往在深夜或海外时段到达、且客户第一诉求是『先看看这东西怎么用』——一个 24/7、多语言、能在真实软件/组态界面里边点边讲的智能体，可在兴趣峰值即时接住海外经销商与工程采购，替代原本要排期数天的远程演示，把稀缺的 SE 资源留给高价值现场谈判。对工业软件/MES/工业 SaaS 供应商尤其直接可迁移；对硬件类产品则受限于『真实产品需在浏览器里可操作』这一前提，更适合其配套的数字平台/组态软件演示。
2. **国内对标厂商**：崤云/主流数字人直播带货与在线客服厂商（能实时语音对话，但多为话术脚本+知识库，缺『真实操作产品界面』的 agentic 能力）、各家 SaaS 的 AI SDR/AI 销售助理（如国内 AI 外呼、会话式获客产品）——覆盖资格审查与跟进，但通常不做产品实操演示、交互式 demo / 产品导览工具的国内对应（Storylane/Navattic 式无代码引导），国内多以自建或轻量 SDK 形式存在，尚无成规模的独立品类玩家
   > 国内暂无与 Karumi 一一对应的『AI 智能体在真实浏览器里实操产品并实时视频演示』独立产品。现有对标散落在三处：数字人客服/直播（强于实时对话、弱于真实操作）、AI SDR 外呼获客（强于资格审查跟进、不做演示）、交互式产品导览（强于引导、非实时智能体）。Karumi 的独特点在于把三者合一并加上『computer-use 式真实操作』，这也是国内下一步可能被复制的空位——尤其结合国产 computer-use/浏览器智能体能力成熟后。
3. **可迁移的干法 / 应进场景词典的词条**：AI demo 智能体、agentic 销售演示、computer-use 浏览器操作、24/7 多语言获客、落地页即时转化、AI 销售工程师、GTM 自动化
   > 可迁移干法：把『兴趣峰值即时接待 + 真实产品实操 + 多语言 24/7』这一组合迁移到任何演示门槛高、SE 稀缺、客户跨时区的 B2B 场景——工业软件、医疗器械配套软件、金融科技后台、企业级 SaaS。核心方法论是用 computer-use 智能体在真实界面里边操作边讲解，替代『录屏 demo』的失真与『真人排期』的延迟，并顺手完成资格审查与 CRM 记录，让转化发生在离兴趣最近的时刻。
4. **风险与存疑点**：①信息稀薄且融资口径冲突——官方未披露融资，第三方聚合站数字（$500K vs $2M）互相矛盾且无投资方核实，极早期公司真实性/规模需谨慎。②护城河存疑：真实浏览器操作产品演示高度依赖底层 computer-use/浏览器智能体能力，一旦通用 agent 能力平台化，该层易被压缩，且 Navattic/Reprise 等在位者与 Naoma 等同代新锐正快速跟进。③交付质量与真实产品耦合风险：智能体需在客户真实 Web 应用里稳定点击/填表，产品改版、复杂交互、演示环境数据准备都可能导致演示翻车，规模化一致性是硬门槛；硬件类/非 Web 产品适配面受限。
