## 转型有术判断
1. **对制造业客户意味着什么**：把 WorkDone 的『监听系统日志 → AI Agent 实时发现异常 → 直接找责任人确认纠错』范式迁到制造业：即在 MES/QMS/ERP 之上架一层实时事件监听 + Agent，专盯质量记录、批记录（batch record）、工艺参数录入的缺漏与不合规，在下道工序/放行/客户审计触发问题前完成闭环纠错——本质是『合规即时防错』而非事后返工。GMP/医疗器械等强监管制造场景尤为契合。
2. **国内对标厂商**：流程自动化/RPA：来也科技（UiBot）、影刀 RPA、艺赛旗、弘玑 Cyclone、流程挖掘：望繁信科技（ABI）、九科信息、垂直医疗合规/病历质控方向国内多为医院 HIS/病案质控系统内嵌模块，尚缺独立『实时 EHR 合规副驾』对标
   > 国内 RPA 三强（来也、影刀、弘玑）已在做流程自动化，但多为『替人执行既定动作』；WorkDone 的差异在『实时监控 + 主动找人纠错』的合规守门人角色，国内可对标望繁信的流程挖掘（发现偏差）叠加 RPA 执行，但『向责任人发起对话即时纠错』的人机协同闭环仍是空白点。
3. **可迁移的干法 / 应进场景词典的词条**：医疗文档合规、实时流程监控、EHR/EMR 集成、保险拒付防控、收入周期管理(RCM)、AI Agent 协同纠错、垂直行业 Agent、康复机构/中小诊所
   > 1) 选高拒付/高返工、且错误源于『录入缺漏』的窄场景切入（对应制造业的质量记录/批记录）；2) 靠标准协议（HL7/FHIR ↔ 制造业的 OPC-UA/MES 接口）低摩擦接入既有系统，不动主系统；3) Agent 载入行业规则（临床协议+付费方规则 ↔ GMP/工艺规程+客户验收标准）；4) 用可量化 ROI 话术（回款+20%/审计-90% ↔ 一次交检合格率、审计工时）撬动付费。
4. **风险与存疑点**：1) 定位漂移/pivot：对外仍挂『business process automation』旧标签，实际已转为医疗合规并启用新品牌 Adentris，判断时须以医疗产品为准。2) 强监管数据：涉 PHI/HIPAA，医疗数据合规与安全门槛高，制造业迁移时对应 GMP 数据完整性（21 CFR Part 11）要求。3) 早期极小：4 人、启动仅 3 家付费诊所、约 5.4 万美元 ARR，客户与营收规模均未经验证，具名客户未披露。4) 效果指标（回款+20%、拒付挽回 15% 等）均为厂商自述，缺第三方验证。5) 深度依赖 EHR 厂商（Kipu/Athena）API，平台方政策变动构成渠道风险。
