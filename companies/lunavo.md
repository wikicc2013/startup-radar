## 转型有术判断
1. **对制造业客户意味着什么**：制造业的物流/发运/采购运营部门有大量与货代同构的'邮件+PDF+ERP/TMS 三处倒腾'的重复工时：到货预约、运单建单、运费对账、在途催单。Lunavo 的'插进现有工具栈+端到端跑后台+只把异常升级给人'的模式，可直接迁移到制造企业的物流运营岗与外贸跟单岗，作为'不增员扩产能'的 AI 数字员工样板。
2. **国内对标厂商**：运去哪（国际货代/跨境物流数字化平台）、YQN 运去那/云通（国际物流数字货代平台）、大件运输与货代 SaaS：oTMS、易流、G7 等物流数字化厂商的运营自动化模块
   > 国内国际物流数字化以运去哪、YQN 为代表做的是'平台撮合+数字货代'，重在交易与履约在线化；Lunavo 走的是'嵌入既有系统、替代后台人力'的 agent 路线，二者互补——国内厂商强在网络与运力，Lunavo 强在单公司后台流程的深度自动化，是可被国内 TMS/货代 SaaS 借鉴的'运营 copilot'方向。
3. **可迁移的干法 / 应进场景词典的词条**：货代后台自动化、订单录入/建单（邮件·PDF·门户解析）、TMS 数据回填、运价报价与调度比价、运费账单稽核、在途状态客服代答、异常处理（滞期/BOL/错送）、AI 数字员工替代运营人力
   > 1) 锚定一个高频且规则清晰的后台流程（如订单录入/运费对账）先做深、拿到'85% 自动化'的硬指标；2) 不替换客户系统，只做执行层插进 TMS/邮件/门户，降低采纳门槛；3) 用'异常才升级+dispatcher 反馈学规则'构建人机协同闭环；4) 用一个真实客户的量化战绩（180+ 单/日、5300+ 邮件/月已自动化）做可信度背书。这套'嵌入式 agent + 单流程深自动化 + 量化战绩'可复用到制造业任何后台运营岗。
4. **风险与存疑点**：1) 团队仅 2 人、公司 2025 年新设，产品与交付成熟度未验证；2) 融资具体金额/估值未公开披露，资金厚度不明；3) 客户仅一家未具名联运货代，商业验证样本极小；4) 德国-旧金山两地+DACH 客群，规模化与美国市场落地存在地域摩擦；5) 货代后台自动化赛道 YC 内外玩家密集，差异化壁垒待观察。
